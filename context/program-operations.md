# Program Operations Context

## Purpose
This file defines the program-level operating structure that supports the institution-level GME office.

## Core Program Roles

### Program Director
Responsible for overall program leadership, educational quality, faculty alignment, and program compliance.

### Associate Program Director
Supports the Program Director with oversight, resident support, curriculum execution, and program operations.

### Program Coordinator
Owns day-to-day program administration, scheduling, communication, documentation, and coordination.

## Program-Level Responsibilities
- Resident onboarding and orientation
- Rotation and schedule coordination
- Evaluation administration
- Faculty and resident communication
- Program documentation
- Accreditation support
- Survey readiness
- Committee and meeting coordination

## Why This Matters
A complete GME office must account for program-level operations, not just institutional functions. Program success depends on clear coordination between institutional leadership and individual programs.

## Coordination with Institutional Office
- Program Directors coordinate with DIO, Associate DIO, and Director of GME
- Program Coordinators coordinate with Accreditation, Resident Affairs, and Faculty Development
- Program leadership depends on Data & Analytics for dashboards and trend visibility
- Program growth or change should route through Strategy & Growth and Finance & Medicare

## Design Standard
Program-level roles should not be treated as informal support functions. They are critical operating roles and should be clearly defined from day 1.
