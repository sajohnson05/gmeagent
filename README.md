# GME AI Operating System

## Purpose
This repository defines an AI-driven operating system for a Graduate Medical Education (GME) Office.

It is designed to:
- Eliminate hidden work
- Ensure accreditation stability
- Protect Medicare funding
- Support residents and faculty
- Enable scalable growth

## Core Components

### Agents
Each agent represents a functional role within the GME office:
- Orchestrator
- Executive
- Associate DIO
- Director of GME
- Finance & Medicare
- Accreditation & Compliance
- Resident Affairs
- Faculty Development
- Data & Analytics
- Strategy & Growth

### Skills
Reusable capabilities used by agents:
- Medicare funding
- Accreditation management
- Faculty development
- Resident operations
- Data analytics
- Strategic planning

### Context
Defines the operating environment and organizational design:
- GME office structure
- Program-level operations
- Operating rhythm
- Interaction model
- Core skill expectations
- Implementation guidance

### Flows
Standard workflows:
- Monthly operations
- Accreditation readiness
- Medicare funding optimization

### Validation
Ensures outputs are:
- Accurate
- Complete
- Actionable

---

## Design Principles
- No hidden responsibilities
- Clear ownership
- Data-driven decisions
- Cross-functional collaboration
- Redundancy in critical knowledge
- Proactive rather than reactive operations

---

## How to Use

1. Load `claude.md` as the core system prompt
2. Use context files to define structure and constraints
3. Route work through the orchestrator
4. Trigger the relevant workflow or agent team
5. Validate the final output for clarity, ownership, and completeness

## Outcome
A fully structured, scalable, and intentional GME operating model powered by AI agents.
