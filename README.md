# GME AI Operating System

## Purpose
This repository defines an AI-driven operating system for a Graduate Medical Education (GME) Office.

It is designed to:
- Eliminate hidden work
- Ensure accreditation stability
- Protect Medicare funding
- Support residents and faculty
- Enable scalable growth

## Core Components

### Agents
Each agent represents a functional role within the GME office:
- Orchestrator (coordination)
- Executive (strategy and escalation)
- Finance & Medicare
- Accreditation & Compliance
- Resident Affairs
- Faculty Development
- Data & Analytics
- Strategy & Growth

---

### Skills
Reusable capabilities used by agents:
- Medicare funding
- Accreditation management
- Faculty development
- Resident operations
- Data analytics
- Strategic planning

---

### Context
Defines the operating environment:
- GME structure
- Data inputs and constraints

---

### Flows
Standard workflows:
- Monthly operations
- Accreditation readiness
- Medicare funding optimization

---

### Validation
Ensures outputs are:
- Accurate
- Complete
- Actionable

---

## Design Principles
- No hidden responsibilities
- Clear ownership
- Data-driven decisions
- Scalable structure
- Proactive operations

## How to Use

1. Load claude.md as system prompt
2. Provide context files
3. Trigger a workflow or request
4. Let orchestrator assign agents
5. Validate output
