# GME AI Operating System

## Purpose
This repository defines an AI-driven operating system for a Graduate Medical Education (GME) Office.

It is designed to:
- Eliminate hidden work
- Ensure accreditation stability
- Protect Medicare funding
- Support residents and faculty
- Enable scalable growth

---

## Getting Started

If you are new, start here:

1. Open: `docs/01-Quick-Start-Claude-Desktop.md`
2. Load `claude.md` into Claude Desktop
3. Use prompts from: `docs/04-Prompt-Library.md`

Example prompt:

Use the GME system to create a faculty development plan for the next quarter. Include ownership, KPIs, and risks.

This system is designed to be used through prompts, not by manually navigating files.

## Core Components

### Agents
Each agent represents a functional role within the GME office:
- Orchestrator
- Executive
- Associate DIO
- Director of GME
- Finance & Medicare
- Accreditation & Compliance
- Resident Affairs
- Faculty Development
- Data & Analytics
- Strategy & Growth

### Skills
Reusable capabilities used by agents:
- Medicare funding
- Accreditation management
- Faculty development
- Resident operations
- Data analytics
- Strategic planning

### Context
Defines the operating environment and organizational design:
- GME office structure
- Program-level operations
- Operating rhythm
- Interaction model
- Core skill expectations
- Implementation guidance

### Flows
Standard workflows:
- Monthly operations
- Accreditation readiness
- Medicare funding optimization

### Validation
Ensures outputs are:
- Accurate
- Complete
- Actionable

---

## Design Principles
- No hidden responsibilities
- Clear ownership
- Data-driven decisions
- Cross-functional collaboration
- Redundancy in critical knowledge
- Proactive rather than reactive operations

## How to Use (Simple)

1. Load `claude.md` into Claude
2. Ask a structured question (use Prompt Library if needed)
3. Let the system route work automatically
4. Review output before acting on regulated decisions
