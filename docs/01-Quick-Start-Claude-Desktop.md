# GME Agent System — Quick Start (Claude Desktop)

## What this is

This system is a pre-built GME operating model.

It includes:
- Defined GME leadership roles (DIO, Program Ops, Faculty Dev, etc.)
- Standard workflows (accreditation, duty hours, grievances, funding)
- Structured decision-making and escalation logic

You will use Claude to activate this system and get structured, role-based output instead of generic answers.

---

## What you need

Before starting, make sure you have:

- Claude Desktop installed
- Access to the `gme-ai-operating-system` folder on your computer

---

## Step 1 — Place the folder

Put the folder somewhere easy to access.

Example:
Documents\GMEAgent\

Inside that folder, you should see:
- claude.md
- agents/
- context/
- flows/
- skills/
- validation/

If you do not see these, stop and fix this before continuing.

---

## Step 2 — Open Claude Desktop

1. Open Claude Desktop
2. Start a new chat

---

## Step 3 — Load the system

Open the `claude.md` file.

Copy everything in that file.

Paste it into Claude as your first message.

This activates the GME operating system.

---

## Step 4 — Confirm it is working

Paste this test prompt:

You are operating as the GME system defined above.

Help me prepare a faculty development plan for the next quarter.
Identify:
- which roles should be involved
- key deliverables
- risks and dependencies

Expected result:
- References Faculty Development, Program Ops, DIO
- Structured deliverables
- Risks identified

---

## Step 5 — Start using it

Examples:
- Run a full accreditation readiness review
- Help me respond to a resident grievance
- Build a faculty development curriculum
- Walk me through a duty hours violation response

---

## Common mistakes

1. Not loading claude.md
2. Asking vague questions
3. Editing system files too early

---

## What success looks like

- Structured answers
- Role-based responses
- Clear workflows
